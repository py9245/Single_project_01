# PRD: Block Navigator Game

## 1. 제품 개요

### 1.1 제품명
Block Navigator (블록 네비게이터)

### 1.2 제품 설명
3개의 블록(Q, W, E)을 개별 또는 동시에 조작하여 움직이는 맵의 벽을 피해 통과하는 횡스크롤 장애물 회피 게임

### 1.3 목표
- 간단한 조작으로 높은 집중력과 반응속도를 요구하는 캐주얼 게임 제작
- 멀티태스킹 능력과 순발력을 테스트하는 게임플레이 제공

## 2. 핵심 기능 요구사항

### 2.1 게임 화면 구성
- **해상도**: 1280x400px (또는 적절한 16:5 비율)
- **배경색**: 회색(#808080) - 이동 가능 영역
- **벽 색상**: 흰색(#FFFFFF) - 충돌 시 게임오버
- **블록 표시**: Q, W, E 각각 다른 색상의 정사각형
  - Q: 노란색(#FFD700)
  - W: 초록색(#00FF00)  
  - E: 파란색(#0080FF)
- **블록 크기**: 30x30px

### 2.2 맵 시스템
- **맵 구성**: 맵1과 맵2가 번갈아가며 무한 반복
- **스크롤 속도**: 초당 50px (조정 가능)
- **맵 이동**: 우측에서 좌측으로 자동 스크롤
- **맵 너비**: 각 맵당 최소 1600px

### 2.3 블록 컨트롤
- **이동 방향**: 상하 이동만 가능 (좌우 고정)
- **이동 속도**: 초당 200px
- **시작 위치**: 화면 좌측 1/4 지점에 세로로 정렬

#### 2.3.1 조작 방법
- **Q키**: Q블록 선택/해제 토글
- **W키**: W블록 선택/해제 토글
- **E키**: E블록 선택/해제 토글
- **방향키 ↑↓**: 선택된 블록 상하 이동
- **복수 선택**: 여러 키를 눌러 동시 선택 가능

#### 2.3.2 선택 상태 표시
- 선택된 블록: 테두리 강조 또는 약간 확대
- 선택되지 않은 블록: 기본 상태

### 2.4 랜덤 강제 이동 시스템
- **발동 주기**: 1초마다
- **대상**: Q, W, E 중 랜덤 1개
- **이동 거리**: 상 또는 하 방향으로 40px
- **예외 조건**: 벽과의 거리가 0.5초(25px) 이내일 때는 발동 안함
- **시각적 피드백**: 강제 이동 시 해당 블록 깜빡임

### 2.5 충돌 판정
- **충돌 대상**: 흰색 벽 영역
- **판정 방식**: 픽셀 단위 충돌 감지
- **결과**: 즉시 게임오버

## 3. UI/UX 요구사항

### 3.1 시작 화면
- 게임 타이틀
- Start 버튼
- 조작법 안내
- 최고 점수 표시

### 3.2 게임 HUD
- **현재 점수**: 생존 시간(초) 또는 통과한 거리
- **선택 상태 인디케이터**: Q/W/E 선택 상태 표시
- **다음 강제 이동 타이머**: 남은 시간 표시 (선택사항)

### 3.3 게임오버 화면
- 최종 점수
- 최고 점수
- Retry 버튼
- Main Menu 버튼

## 4. 기술 요구사항

### 4.1 개발 환경
- **플랫폼**: 웹 브라우저 (Chrome, Firefox, Safari, Edge 최신 버전)
- **프레임워크**: HTML5 Canvas 또는 React 기반
- **언어**: JavaScript/TypeScript

### 4.2 성능 요구사항
- **목표 FPS**: 60fps 유지
- **입력 지연**: 50ms 이하
- **메모리 사용**: 100MB 이하

## 5. 게임 밸런싱

### 5.1 난이도 조절
- **초급**: 맵 스크롤 속도 느림, 강제 이동 빈도 낮음
- **중급**: 기본 설정
- **고급**: 맵 스크롤 속도 빠름, 강제 이동 빈도 높음

### 5.2 점수 시스템
- 기본 점수: 시간당 100점
- 보너스: 강제 이동 회피 성공 시 +50점

## 6. 개발 우선순위

### Phase 1 (MVP)
1. 기본 블록 3개 구현
2. 개별/복수 선택 시스템
3. 맵 스크롤링
4. 충돌 판정
5. 게임오버 처리

### Phase 2
1. 랜덤 강제 이동 시스템
2. 점수 시스템
3. UI/HUD 구현

### Phase 3
1. 시작/종료 화면
2. 사운드 효과
3. 난이도 선택
4. 최고 점수 저장

## 7. 테스트 요구사항

### 7.1 기능 테스트
- 각 블록의 독립적 컨트롤
- 복수 선택 시 동시 이동
- 충돌 판정 정확도
- 랜덤 이동 타이밍

### 7.2 성능 테스트
- 프레임 드롭 체크
- 메모리 누수 확인
- 입력 반응성

## 8. 제약사항
- 블록은 좌우 이동 불가 (상하만 가능)
- 맵 스크롤 속도는 게임 중 변경 불가
- 블록이 화면 상하 경계를 벗어날 수 없음
